```mermaid
graph TD
    A[ผู้ใช้งาน] --> B(เบราว์เซอร์);
    B --> C{เข้าถึง index.php};
    C -- ไม่ได้ล็อกอิน --> D[login.php];
    C -- ล็อกอินแล้ว --> E[index.php];

    D -- ฟอร์มลงทะเบียน --> F{login.php (จัดการลงทะเบียน)};
    F -- บันทึก/อัปเดต --> G[users.json];
    F -- สำเร็จ --> D;
    F -- ล้มเหลว --> D;

    D -- ฟอร์มล็อกอิน --> H{login.php (จัดการล็อกอิน)};
    H -- อ่านจาก --> G;
    H -- สำเร็จ --> I(Session);
    I --> E;
    H -- ล้มเหลว --> D;

    E -- คลิกออกจากระบบ --> J[logout.php];
    J -- ทำลาย --> I;
    J --> D;

    subgraph PHP Server
        D; F; H; J; E; G; I;
    end
```

**คำอธิบาย Flowchart:**

*   **ผู้ใช้งาน (User)**: ผู้ที่ใช้งานระบบ
*   **เบราว์เซอร์ (Browser)**: โปรแกรมที่ผู้ใช้ใช้เข้าถึงเว็บไซต์
*   **เข้าถึง index.php**: จุดเริ่มต้นเมื่อผู้ใช้พยายามเข้าถึงหน้าหลัก
*   **login.php**: หน้าที่จัดการการล็อกอินและการลงทะเบียน
*   **index.php**: หน้าหลักของแอปพลิเคชันที่ต้องล็อกอินก่อนเข้าถึง
*   **users.json**: ไฟล์ที่เก็บข้อมูลผู้ใช้ (ชื่อผู้ใช้และรหัสผ่านที่ถูกแฮช)
*   **Session**: กลไกการจัดการสถานะการล็อกอินของผู้ใช้บนเซิร์ฟเวอร์
*   **logout.php**: หน้าที่จัดการการออกจากระบบ
